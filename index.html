How to avoid Apple revoking?

* This guide is only for devices that have been certified for trusted app in Settings and can still open app. You need to install zinitevi app and trust certificate from Settings of your device before enabling dnscloak, that mean when you are using app normally and not protected by dnscloak, one day app gives a "Unable to Verify App" message then Apple revoked this certificate, what you need to do:

  • Delete app and wait for it to be signed by new certificate and now install dnscloak and enable it.
  • Reset to factory device and reinstall app with dnscloak is enabled.

1. Disable proxy on your iOS device, go to Settings / touch on WiFi / touch "i" next to the WiFi connection / scroll down to HTTP PROXY / set Configure Proxy to Off.

2. Download DNSCloak from AppStore

3. Download AppleRevoking.txt to your iOS device.

4. Open DNSCloak app

5. Tap on menu in the left corner. In DNSCloak Settings turns on "Connect on Demand".

6. Turn on "Enable Blacklist", a file menu will open and select the earlier downloaded file "AppleRevoking.txt"

7. Go back to main app and select "a-and-a" / "Use this server"

* You need to trust the app before doing this or tap "Stop using this server" / trust app from Settings / "Use this server"

Please check the following steps to know it's working perfectly

  • Your iOS device Open Settings / General / VPN & Device Management / VPN / make sure VPN is Enable from DNSCloak app, touch "i" on DNSCloak row / enable "Connect On Demand"

  • Open safari / enter ppq.apple.com, make sure black page with text: Safari cannot open the page because the server cannot be found.

8. Now swipe up to close DNSCloak app to keep it running on background

* You can close DNSCloak but must keep VPN connection.
